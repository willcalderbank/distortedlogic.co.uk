While building this site it became clear that a database based CMS was overkill, thats when I discoverd Jekyll a very nice 'offline' CMS which with the use of templates will "Transform your plain text into static websites and blogs".

A static site comes with a number of benifits manily performance and that you dont have to worry about a database, but also has its downsides. Editing posts becomes much harder, thankfully with Jekyll, Amazon S3, s3_website and a little upload script this downside becomes less of problem. This script can be run automaticly when the site is pushed to github, which is very nice.

First install Jekyll and s3_website next you create a new repo on github and clone it, now in that directory run:

{% highlight bash %}s3_website cfg create {% endhighlight %}

This will create a s3_website.yaml file, fill it out with all your setting like s3_website suggests. Remember to add this file to .gitignore or use enviroment variables!
